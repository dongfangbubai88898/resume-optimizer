## 6. 技术总结

- **项目架构：** Express + 静态文件服务，所有 API 调用直接走 DeepSeek API
- **依赖：** 仅有 `express`（通过 npm 安装），无其他后端框架
- **API Key 安全：** 从环境变量读取，代码中无硬编码
- **本地启动命令：** `DEEPSEEK_KEY=sk-xxx node server.js`
- **Node.js 版本：** v24.14.0（兼容良好）

### 建议
1. 如果要长期运行本地服务，考虑使用 `pm2` 或 `systemd` 托管进程
2. 搜索功能（bocha）如需启用，建议申请其免费额度
3. 当前 optimize 接口返回的 `optimizedResume` 是结构化 JSON 而不是纯文本简历，前端需适配展示
